SnackBox Routing Spec

SnackBox uses strict routing with explicit paths and a clear 404 for anything unmatched.

Rules

  1. / → Landing
  2. /public/* → Static files (no traversal)
  3. /search → Local index search (TSV-backed)
  4. /docs → Docs index
  5. /docs/:slug → Render a specific document from data/docs/:slug.html

404 Behavior

Any non-matching path returns a minimal HTML 404 page that includes the rejected target path.

Why strict?

  • Predictable behavior
  • Security (no accidental fallthrough)
  • Faster debugging
